# MyTaxs.online 深度诊断与改进建议报告

> 生成日期:2026-06-19 · 方法:5 路并行网络研究(Budget 2026 税改 / e-Invoice 2026 / 竞品 / 纳税人痛点 / 工具站最佳实践)+ 逐项对照本仓库实际代码。
> 说明:多数官方/事务所页面对自动抓取返回 403,法规结论基于多源搜索摘要交叉验证(KPMG/PwC/EY/Crowe/LHDN/RinggitPlus 等),核心数字均有 2+ 独立来源。**涉及金额的改动,落地前建议再核对一次 LHDN 官方公报。**

---

## 总览结论

- **你的计算器广度已领先全马同类**:公司税、SST(含 2025 扩围)、印花税、批量 PCB、预扣税、资本免税额、独资税、AI 助手、e-Invoice 阶段查询——这些竞品(RinggitPlus/iMoney/ringgitcalc/CukaiMax)大多没有,是真实差异化优势。
- **真正的短板有三块**:① 计算引擎里有**几处过期/缺失的数字与年度**;② 缺**“判定/规划”类交互工具**(竞品的高价值方向);③ 缺**内容生态 + SEO 基建**(竞品的流量护城河)。
- **e-Invoice 引擎已是最新**(RM1m 豁免线、取消 Phase 5、Phase 4 宽限至 2027),这点做得很好。

## 一、🔴 准确性问题(代码里的过期数字)— 最高优先,直接影响可信度

税务工具的第一生命线是“算得对”。以下是研究与代码对照后发现的**确凿/疑似过期项**:

### 1. 残障类减免金额过期(YA2025 已上调,代码仍是旧值)
`src/engine/tax-reliefs.ts`:

| relief id | 代码现值 | YA2025 应为 | 置信度 |
|---|---|---|---|
| `disabled_individual` | 6000 | **7000** | 高 |
| `disabled_spouse` | 5000 | **6000** | 中高 |
| `disabled_child` | 6000 | **8000** | 高 |

> 依据:Budget 2025(YA2025 生效)上调残障人士相关减免。来源 ClearTax/Crowe/EY Budget 2025、Human Resources Online。

### 2. 疑似过期 / 待核实
- `education_medical_insurance` 代码为 3000 → 有来源称 YA2025 上调至 **4000**(中置信度,**务必核对 LHDN 后再改**)。
- “学习障碍儿童”诊断/早疗子减免(属医疗大类)YA2025 由 4000→**6000** — 代码无此独立项。
- `housing_loan_interest` 7000:注意 Budget 2025 新增的“**首套自住房贷款利息减免**”(2025–2027 签 SPA:≤RM500k 房产 RM7,000 / RM500k–750k RM5,000)是**另一项分级新政**,与现有项不同,建议单列。

### 3. 税率档无需改动 ✅
`src/engine/tax-rates.ts` 的 YA2025 税阶正确;研究确认 YA2025/YA2026 **个人税率档无变化**(Budget 2026 “无新税、仅扩大减免”)。

## 二、🟠 计算引擎的内容更新

### 4. 年度参数形同虚设(结构性)
`getReliefDefinitions(year)` 和 `getTaxRates(year)` 都**忽略入参、永远返回 YA2025**。一旦要支持 YA2026 申报(2027 年报税季),需要真正按年份分支。建议现在就把数据结构改成 `Record<year, ...>`。

### 5. 缺 YA2026 减免集(Budget 2026 已公布)
为下一报税季预留。Budget 2026 新增/扩大(YA2026 生效):
- **新增**:本地旅游景点/文化演出门票减免 **RM1,000**(仅 YA2026)。
- **扩大**:疫苗减免扩到所有 NPRA 注册疫苗;托儿减免 RM3,000 永久化 + 范围扩至 12 岁以下注册日托/课后中心;RM2,500 绿色家居减免新增厨余粉碎机等(YA2026–2027 一次性)。

### 6. 新“股息税”可能未建模(需复核)
`src/engine/personal.ts` 第 12 行把 `dividend` 直接计入累进应税收入。但马来西亚个人股息为单层制(一般免税),**Budget 2025 新增的是对个人股息收入超过 RM100,000 部分征 2%**(YA2025,独立于累进税)。建议复核:① 普通股息是否应免于累进;② 是否需要单独的“2% 股息税”计算。**(中置信度,先核实再动)**

### 7. e-Invoice 引擎已最新,可加两条 UI 提示(非数字错误)
`src/engine/e-invoice.ts` 数据正确。可在结果里补充展示:
- **RM10,000 单笔规则**(2026-01-01 起,单笔 ≥ RM10k 必须逐笔开,不能合并)——代码注释已有,建议显示给用户。
- **关联方反豁免**:营业额 < RM1m 但属于 ≥ RM1m 的集团/有大股东母公司,则**不享豁免**。
- 把“强制日 2026-01-01”与“Phase 4 处罚执法 2028-01-01(宽限至 2027-12-31)”分开说明,避免用户误以为 2026 就会被罚。

### 8. 公司税:可考虑的进阶项(非错误)
`corporate.ts` 的 SME 三档与资格判定(股本≤2.5M、营收<50M、外资>20% 失格、关联公司)均正确。可选增补:**资本利得税(CGT,未上市股权处置 10%,2024-03 起)**、**全球最低税(15%,大型跨国)**——属高阶,优先级低。

## 三、🟡 功能缺口(对比竞品的高 ROI 方向)

竞品(CukaiMax RM96/年、CukaiMate、RinggitWise、ringgitcalc)已把“计算器”升级为“**判定 + 规划 + 退税**”。你已有的判定向导几乎为零,这是最大产品机会。

### 高优先(ROI 最高,且市场尤其缺中文/马来语版本)
1. **「我需要报税吗 / 用哪张表(BE/B/M)」判定器** — 消解全马最高频误解“报税≠缴税”;天然获客入口。
2. **「配偶合并 vs 分开报税」对比计算器** — 一键算两种方案税额与净省额。现有 `income-form` 仅处理 spouse 减免,**没有方案对比**;官方都建议“两种都算一遍”,正是工具价值点。
3. **退税估算器 + 漏报减免扫描** — 对比已扣 PCB vs 实际应税 → 预测退/补税;按用户情况(已婚/有孩/买房/保险/医疗)逐项提示可申报减免。直接“帮用户省钱/退钱”,转化力最强。
4. **e-Invoice「我要做吗 + 何时 + consolidated/self-billed 怎么分」判定器** — 你已有阶段查询,升级为决策向导;法规高频变动 = 持续流量。

### 中优先
5. **SME 优惠税率资格判定器 + SST 注册判定器**(行业 + 营业额 → 是否注册、6%/8%、应税/豁免)。
6. **资本免税额计算器**:你已有 `capital-allowance-calculator.tsx`,建议给独立 URL + 面向独资/SME 的逐年折旧表说明(几乎无对手)。
7. **Freelancer/Gig 报税向导**:Form B、CP500 分期、SOCSO 自雇(Gig Workers Act 2025,2026-03-31 全面生效)提醒——增长群体、被忽视。
8. **CP204 安全估算 + 第 6/9 月修订截止提醒**(避开“低估超 30% 罚 10%”陷阱;新 SME 首两年是否要交零估)。
9. **可补的专项计算器**:租金收入税(含 quit rent/利息/维修抵扣)、非居民/外籍统一税率、奖金/第13月薪净额、津贴应税判定。

### 较大投入(差异化但工程量大)
10. **收据/文件管理 + OCR 自动归类**(CukaiMax/CukaiMate/RinggitWise 的主战场)。
11. **多年税务并排对比 / YoY 趋势**(几乎全行业空白)。
12. **PDF 报告导出**(带 logo,可交会计师/银行)——天然付费点,见第六节。

## 四、🟡 内容生态(竞品最强护城河,你最薄弱)

RinggitPlus/iMoney 靠**报税教程 + 减免清单 + 章节化指南**拿走大量 SEO 流量。你已有 `efiling-guide.tsx` 组件和 3 篇三语博客,基础不错,但体系化不足。建议补:
1. **年度 e-Filing 分步图文指南**(数字证书激活、MyTax 首登、e-BE/e-B 表单选择、签署提交)。
2. **「全部可申报减免」权威清单(YA2025 + Budget 2026)**,由税务专业人士审核署名。
3. **可下载/可分享的减免 Checklist 信息图**(RinggitPlus 拳头内容)。
4. **Budget 预算案税务变动解读专题**(每年更新)。
5. 把社群 9 大高频问题做成 **博客系列 + AI 助手意图库**:① 我要报税吗 ② 副业/freelance 怎么报 ③ 合并 vs 分开 ④ 能扣哪些/漏了啥 ⑤ 退税为何没到 ⑥ 海外收入(FSI)要不要报 ⑦ 报错怎么改 ⑧ 小生意要不要 e-invoice ⑨ 第一次报税完整步骤。

> **最大差异化点**:真正的**三语(EN/中文/BM)交互式判定工具**——市场上中文/马来语几乎只有博客、没有工具。把“判定逻辑”做成三语工具 + 三语 AI 助手,是当前近乎空白的蓝海。

## 五、🟢 SEO / 技术基建(代码已确认的缺口)

1. **工具拆分独立 URL**:`corporate-tools-tabs.tsx` / `property-tools-tabs.tsx` 把多个工具(RPGT、印花税、资本免税额、CP204、预扣税)塞在同一 URL 的 tab 里,不利收录。应各给独立可索引 URL(如 `/rpgt-calculator`、`/stamp-duty-calculator`),各配独立 H1/title/description。
2. **计算器页加结构化数据**:目前只有博客用了 `Article` schema。每个工具页应加 `WebApplication`(`applicationCategory: "FinanceApplication"`)+ `FAQPage` schema。
3. **`sitemap.ts` 的 `lastModified: new Date()`** 恒为构建当天,Google 会不信任;应用内容真实修改日期。
4. **博客 E-E-A-T(税务属 YMYL 最高敏感类)**:`blog/[slug]/page.tsx` 的 `author` 是 `Organization` 无真人资质;schema 缺 `dateModified`。应改真人作者 + 资质(LHDN tax agent / ACCA / CTIM)+ `Person` schema + “Reviewed by”。
5. **内部链接**:博客 CTA 目前链回首页 `/`,应改为链到**对应计算器页**;工具页 ↔ 相关博客互链。

## 六、🟢 变现(你已是 Stripe 积分 freemium,可优化)

1. **付费墙时机**:让用户**先看到完整计算结果**,在“导出 PDF / 批量 / AI 深度分析”那一步才扣积分(先见价值再付费,转化更高)。不要进页面就挡。
2. **选对付费点**:核心单笔计算永久免费(引流+SEO+信任);把 **PDF 报告导出、批量处理(你已有 `batch-pcb`)、AI 多轮深度咨询、历史/多公司管理**设为付费。
3. **定价页加锚定/诱饵**:`pricing-cards.tsx` 现为 3 个等价积分包;建议中间包标“最受欢迎”、加一个高价诱饵包,提升中高阶选择。
4. **积分 + 订阅混合**:散户用积分;会计师/SME 重度用户给月/年订阅(无限批量 + 年度更新)。

## 七、🟢 信任 / 合规

1. **`SourceNote` 全站铺开**:目前只在首页、corporate、employer 三处用了;应给**所有计算器**(SST、property、e-invoice、batch-pcb 等)都挂上官方来源 + 验证日期。这是你已有的好资产,利用率不足。
2. **“数据不保存”承诺**:计算引擎是客户端 TS,理论上收入数字不必上传。**先在代码层确认确实不上传 PII**,再在每个计算器页放醒目标语“你输入的收入数据不会被保存或上传”——税务工具最强的信任/转化信号。
3. **隐私页升级到 PDPA 2024 修正案(2025-06-01 生效)**:`privacy/page.tsx` 结构完整,但需补:DPO 联系方式、强制数据泄露通报承诺、用户权利(访问/更正/可携/删除)、Stripe/Supabase 跨境处理者披露、数据保留期限。
4. **AI 助手信任**:每次对话界面**持续显示**免责提示(非仅登录页);每条回答附**可点击官方引用**(RAG 已有基础);不确定时主动承认并导向计算器或 LHDN 官方页。
